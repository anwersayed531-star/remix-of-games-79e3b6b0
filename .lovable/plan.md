

# إضافة مسح QR Code بالكاميرا لنظام الاتصال بالأكواد

## المشكلة
حالياً اللاعب يضطر ينسخ كود طويل (60-80 حرف) ويلصقه يدوياً — تجربة صعبة خصوصاً على الموبايل. الحل: عرض الكود كـ QR Code + إتاحة مسحه بالكاميرا.

## التغييرات

### 1. إضافة مكتبات QR
- `qrcode.react` — لتوليد صورة QR من الكود
- `html5-qrcode` — لمسح QR بكاميرا الجهاز (تعمل على الويب والأندرويد بدون بلاجن أصلي)

### 2. مكون عرض QR Code (`src/components/QRCodeDisplay.tsx`)
- يأخذ النص (الكود) ويعرضه كصورة QR
- تحته زر "نسخ الكود" الموجود حالياً كخيار بديل
- حجم مناسب للموبايل مع خلفية بيضاء للوضوح

### 3. مكون ماسح QR (`src/components/QRCodeScanner.tsx`)
- يفتح الكاميرا ويمسح QR Code
- يطلب إذن الكاميرا تلقائياً
- عند المسح الناجح يُرجع النص ويغلق الكاميرا
- زر إغلاق + رسالة خطأ بالعربي إذا الكاميرا مش متاحة
- خيار بديل: "أدخل الكود يدوياً" في حالة فشل الكاميرا

### 4. تحديث `MultiplayerLobby.tsx`
- **المضيف**: عند عرض كود الغرفة (offer) → يعرضه كـ QR Code بدل النص الطويل + زر نسخ تحته
- **الضيف (انضمام)**: بدل حقل النص فقط → زر "مسح QR" يفتح الكاميرا + حقل النص كبديل
- **المضيف (انتظار الرد)**: عند طلب كود الرد → زر "مسح QR" + حقل لصق
- **الضيف (عرض كود الرد)**: يعرض answer code كـ QR Code

### 5. إضافة إذن الكاميرا في `register-plugin.cjs`
- إضافة `android.permission.CAMERA` في AndroidManifest.xml

## ملاحظات
- `html5-qrcode` تستخدم الكاميرا عبر Web API (`getUserMedia`) — تعمل في Capacitor WebView بدون بلاجن أصلي
- الحقل اليدوي (نسخ/لصق) يبقى متاح دائماً كبديل
- التصميم متوافق مع الثيم الحالي (gold/wood-texture)

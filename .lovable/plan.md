

# تحسين نظام الاتصال: كود ثابت للجهاز + مشاركة + حفظ الاتصال بين الألعاب

## ملخص الطلبات
1. إدخال الكود يدوياً يظهر دائماً بجوار كل وسيلة اتصال
2. مشاركة الكود عبر البلوتوث/التطبيقات (Web Share API)
3. كود ثابت لكل جهاز (Device ID) يُنشأ مرة واحدة ويُحفظ في localStorage
4. الاتصال يبقى بين الألعاب بدون إعادة اتصال (Global Connection Context)
5. لا مدة انتهاء على الأكواد

## التغييرات

### 1. كود الجهاز الثابت — `src/lib/deviceCode.ts`
- توليد كود قصير (6 أحرف) عشوائي مرة واحدة وحفظه في `localStorage`
- دالة `getDeviceCode()` ترجع نفس الكود دائماً
- دالة `getDeviceName()` / `setDeviceName()` لحفظ اسم اللاعب

### 2. زر مشاركة الكود — تحديث `QRCodeDisplay.tsx`
- إضافة زر "مشاركة" يستخدم `navigator.share()` (Web Share API)
- يعمل على أندرويد ويفتح قائمة المشاركة (بلوتوث، واتساب، إلخ)
- إذا `navigator.share` غير متاح، يبقى زر النسخ فقط

### 3. إدخال يدوي دائم — تحديث `MultiplayerLobby.tsx`
- في كل مكان يوجد QR أو كاميرا، يظهر حقل الإدخال اليدوي بجواره مباشرة (ليس مخفي خلف زر)
- الـ QR + الكاميرا + الحقل اليدوي كلهم ظاهرين معاً

### 4. اتصال مستمر بين الألعاب — `src/contexts/ConnectionContext.tsx`
- React Context يلف التطبيق بالكامل
- يحتفظ بالـ `useMultiplayerSync` instance واحد مشترك
- عند الانتقال بين الألعاب، الاتصال لا ينقطع
- كل لعبة تستخدم `useConnection()` بدل إنشاء اتصال جديد

### 5. تحديث `App.tsx`
- لف التطبيق بـ `ConnectionProvider`

### 6. عرض كود الجهاز في الإعدادات — تحديث `SettingsDialog.tsx`
- عرض كود الجهاز الثابت مع زر نسخ
- إمكانية تغيير اسم اللاعب

## ملاحظات
- Web Share API يدعم المشاركة عبر بلوتوث وكل التطبيقات على أندرويد
- كود الجهاز مختلف عن كود الغرفة (SDP) — كود الجهاز للتعريف فقط
- أكواد SDP لا تنتهي طالما التطبيق مفتوح (لا timeout)
